# 🔧 Chrome 代理配置问题（2025-11-26）

## ❌ 问题

- Chrome 提示"错误的URL，无网络连接"
- 说明 Chrome 没有使用系统代理设置

---

## ✅ 解决方案

### 方法1：重新配置 Wi-Fi 代理（最重要）

Chrome 使用系统 Wi-Fi 代理，需要确保代理配置正确：

#### 步骤：

1. **打开设置 → Wi-Fi**
2. **点击你连接的 Wi-Fi 网络**（点击 ⓘ 图标）
3. **向下滚动到"HTTP 代理"**
4. **检查配置**：
   - 选择"手动"
   - 服务器：`47.243.177.166`（确保没有空格）
   - 端口：`3128`（确保是数字）
   - 认证：关闭
5. **保存并返回**

6. **重新连接 Wi-Fi**：
   - 关闭 Wi-Fi（等待 5 秒）
   - 重新打开并连接

7. **测试**：
   - 先访问 `http://www.baidu.com`（测试代理是否工作）
   - 如果能打开，再访问 `https://www.google.com`

### 方法2：清除 Chrome 缓存

1. **打开 Chrome**
2. **点击右下角三个点**（菜单）
3. **设置 → 隐私设置和安全性**
4. **清除浏览数据**
5. **选择"全部时间"**
6. **清除数据**

### 方法3：使用 Safari 测试

既然 Hotmail 显示在香港登录，说明 Safari 可能工作：

1. **打开 Safari**
2. **清除缓存**：设置 → Safari → 清除历史记录和网站数据
3. **访问**：`https://accounts.google.com/signup`
4. **看是否能打开**
