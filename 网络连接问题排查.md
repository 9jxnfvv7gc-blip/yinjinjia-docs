## 🔍 错误分析

**错误信息**：`Connection failed (OS Error: No route to host, errno = 65)`

**可能的原因**：
1. **iPhone 没有连接到网络**：WiFi 或移动网络未连接
2. **网络路由问题**：iPhone 无法路由到服务器
3. **防火墙/安全组**：服务器防火墙或安全组阻止了连接
4. **服务器问题**：服务器可能暂时不可用

## 🔧 解决网络连接问题

### 方法 1：检查 iPhone 网络连接

1. **确保 iPhone 已连接到网络**：
   - 打开 `设置 → WiFi`，确认已连接 WiFi
   - 或确认移动数据已开启

2. **测试网络连接**：
   - 在 iPhone 的 Safari 浏览器中访问：`http://47.243.177.166:8081`
   - 如果能看到服务器页面，说明网络正常

### 方法 2：在应用中重试

1. **点击"重试"按钮**：应用界面上的重试按钮
2. **下拉刷新**：在应用界面下拉刷新
3. **点击"重新连接"按钮**：如果有的话

### 方法 3：检查服务器状态

在 Mac 终端执行：
```bash
# 测试服务器是否可访问
curl http://47.243.177.166:8081/api/categories
```

如果返回 JSON 数据，说明服务器正常。
