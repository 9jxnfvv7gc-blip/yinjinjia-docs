# 空应用上架和后续添加内容方案

## ✅ 完全可以先上架空应用

### 答案：可以！

**完全可以先上架空应用（没有视频和歌曲），通过审核后再添加内容。**

---

## 📋 上架流程

### 步骤1：部署服务器（即使没有内容）

#### 1.1 在服务器上创建目录

在服务器Workbench终端执行：

```bash
mkdir -p /root/videos/原创视频
mkdir -p /root/videos/原创歌曲
mkdir -p /root/video_server
```

---

#### 1.2 上传并配置video_server.py

**使用Workbench文件上传**：
1. 上传 `video_server.py` 到 `/root/video_server/`
2. 修改配置：

```bash
cd /root/video_server
sed -i 's|VIDEO_ROOT = "/Volumes/Expansion"|VIDEO_ROOT = "/root/videos"|g' video_server.py
```

---

#### 1.3 启动服务器

```bash
cd /root/video_server
python3 video_server.py
```

**即使没有内容，服务器也可以正常运行**，只是返回空列表。

---

### 步骤2：测试空应用

#### 2.1 运行应用

```bash
cd "/Volumes/Expansion/FlutterProjects/桌面影音播放器_安装包_20251121_165905"
flutter run -d macos
```

**应该看到**：
- ✅ 应用正常打开
- ✅ 显示"原创视频：0个"和"原创歌曲：0个"
- ✅ 显示"还没有内容"的提示
- ✅ 可以点击"上传新内容"按钮

---

### 步骤3：构建并上架

#### 3.1 iOS（App Store）

```bash
open ios/Runner.xcworkspace
```

在Xcode中：
1. 配置签名（选择你的Apple开发者账号）
2. Product → Archive
3. Distribute App → App Store Connect
4. 上传并提交审核

---

#### 3.2 Android

```bash
# 在服务器上构建
cd /root/app
source ~/.bashrc
flutter build apk --release
```

---

### 步骤4：通过审核后添加内容

#### 方法1：在应用中上传

1. **用户打开应用**
2. **点击"上传新内容"按钮**
3. **选择视频或音乐**
4. **上传到服务器**

---

#### 方法2：手动上传到服务器

```bash
# 上传视频到服务器
rsync -avz --progress /本地路径/视频文件.mp4 root@47.243.177.166:/root/videos/原创视频/

# 上传歌曲到服务器
rsync -avz --progress /本地路径/歌曲文件.mp3 root@47.243.177.166:/root/videos/原创歌曲/
```

**或使用Workbench文件上传**
