# 🔧 豆包访问问题解决步骤（2025-11-26）

## ❌ 问题

浏览器打不开豆包网站，但网络连接正常（ping 和 curl 都能访问）。

---

## 🔍 原因分析

虽然代理显示 "Enabled: No"，但系统代理设置中仍保留着 Clash Verge 的地址（127.0.0.1:7890），这可能导致浏览器尝试连接已关闭的代理服务器。

---

## ✅ 解决步骤

### 步骤1：完全清除代理设置

我已经在终端执行了清除命令。如果还不行，请手动操作：

1. **打开"系统设置"**（System Settings）
2. **点击"网络"**（Network）
3. **选择"Wi-Fi"**
4. **点击"详细信息"**（Details）
5. **点击"代理"**（Proxies）标签
6. **取消勾选所有代理选项**：
   - ☐ Web 代理（HTTP）
   - ☐ 安全 Web 代理（HTTPS）
   - ☐ SOCKS 代理
7. **点击"好"**（OK）

### 步骤2：清除 DNS 缓存

在终端运行（需要输入密码）：

```bash
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

### 步骤3：清除 Safari 缓存

1. **打开 Safari**
2. **Safari → 设置**（Preferences）
3. **点击"隐私"**（Privacy）标签
4. **点击"管理网站数据"**（Manage Website Data）
5. **点击"全部移除"**（Remove All）
6. **确认删除**

### 步骤4：重启 Safari

1. **完全退出 Safari**（Command + Q）
2. **等待 5 秒**
3. **重新打开 Safari**
4. **访问**：`https://www.doubao.com`

### 步骤5：如果还是不行，尝试其他浏览器

1. **打开 Chrome**（如果已安装）
2. **访问**：`https://www.doubao.com`
3. **看是否能打开**

---

## 🔍 检查代理是否已清除

在终端运行：

```bash
networksetup -getwebproxy "Wi-Fi"
networksetup -getsocksfirewallproxy "Wi-Fi"
```

如果显示 "Enabled: No" 且没有 Server 地址，说明代理已清除。

## ✅ 问题已解决

通过完全清除系统代理设置（包括清除代理服务器地址），豆包网站已能正常访问。

**解决方法**：
```bash
networksetup -setwebproxystate "Wi-Fi" off
networksetup -setsecurewebproxystate "Wi-Fi" off
networksetup -setsocksfirewallproxystate "Wi-Fi" off
networksetup -setwebproxy "Wi-Fi" "" 0
networksetup -setsecurewebproxy "Wi-Fi" "" 0
networksetup -setsocksfirewallproxy "Wi-Fi" "" 0
```
