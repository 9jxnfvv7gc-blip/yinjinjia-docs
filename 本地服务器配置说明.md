# 本地服务器配置说明

## 问题

本地服务器启动时使用的视频根目录是 `/root/videos`，但这个路径在 Mac 上不存在，导致无法找到视频文件。

## 解决方案

### 方法 1：使用环境变量启动（推荐）

```bash
cd "/Volumes/Expansion/FlutterProjects/桌面影音播放器_安装包_20251121_165905"

# 设置正确的视频根目录
export VIDEO_ROOT="/Volumes/Expansion"

# 启动服务器
python3 video_server.py
```

### 方法 2：使用启动脚本

运行我创建的启动脚本：

```bash
cd "/Volumes/Expansion/FlutterProjects/桌面影音播放器_安装包_20251121_165905"
./启动本地服务器（正确路径）.sh
```

### 方法 3：直接指定路径

修改 `video_server.py` 的默认路径（不推荐，因为会影响服务器部署）

## 目录结构

确保视频目录结构如下：

```
/Volumes/Expansion/
├── 原创视频/
│   ├── 视频1.mp4
│   ├── 视频2.mp4
│   └── ...
└── 原创歌曲/
    ├── 歌曲1.mp3
    ├── 歌曲2.mp3
    └── ...
```

## 验证

启动服务器后，在浏览器访问：
```
http://192.168.10.103:8081/api/list/原创视频
```

应该能看到视频列表（如果有视频文件的话）。

## 注意事项

1. **保持服务器运行**：启动服务器的终端窗口必须保持打开
2. **确保目录存在**：确保 `/Volumes/Expansion/原创视频` 和 `/Volumes/Expansion/原创歌曲` 目录存在
3. **网络要求**：Mac 和 iOS 设备必须在同一 WiFi 网络
