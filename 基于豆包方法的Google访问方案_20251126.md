- **Ubuntu 用 Clash for Linux 或 3Proxy**
- **iPhone 配置 Wi-Fi 代理**

## 🚀 推荐方案：Mac 作为中转代理

### 方案1：Mac 安装 ClashX Pro（推荐）

#### 步骤1：在 Mac 上安装 ClashX Pro

1. **下载 ClashX Pro**：
   - 访问：https://github.com/yichengchen/clashX/releases
   - 下载最新版本（.dmg 文件）
   - 安装到 Mac

#### 步骤2：配置 ClashX Pro 连接香港服务器

1. **获取服务器代理配置**：
   - 如果你有 V2Ray/Shadowsocks 配置，导入到 ClashX Pro
   - 或使用现有的代理服务

2. **测试连接**：
   - 在 Mac 上打开 ClashX Pro
   - 选择香港节点
   - 测试访问 `https://www.google.com`
   - 确认可以访问

#### 步骤3：开启局域网共享

1. **打开 ClashX Pro 偏好设置**：
   - 点击菜单栏的 ClashX Pro 图标
   - 选择"偏好设置"

2. **开启局域网连接**：
   - 勾选"允许局域网连接"
   - 记录 HTTP 端口（通常是 7890）
   - 记录 SOCKS5 端口（通常是 7891）

3. **查看 Mac 内网 IP**：
   ```bash
   ipconfig getifaddr en0
   ```
   - 记录这个 IP（例如：192.168.1.105）

#### 步骤4：配置 iPhone Wi-Fi 代理

1. **打开 iPhone 设置 → Wi-Fi**
2. **点击当前连接的 Wi-Fi 网络**（点击 ⓘ 图标）
3. **向下滚动到"HTTP 代理"**
4. **选择"手动"**
5. **填写**：
   - 服务器：Mac 的内网 IP（例如：192.168.1.105）
   - 端口：7890（HTTP 端口）
   - 认证：关闭
6. **保存并返回**

#### 步骤5：测试连接

1. **在 iPhone 上打开 Safari**
2. **访问**：`http://ipinfo.io`
3. **确认 IP 已切换**（应该显示香港或外国 IP）
4. **访问**：`https://accounts.google.com/signup`
5. **看是否能打开**

如果返回 200 或 302，说明代理可以访问 Google。

#### 步骤2：Mac 安装并配置代理转发工具

如果 Mac 可以直接访问 Google（通过 VPN 或其他方式），可以：

1. **安装代理转发工具**（如 3Proxy 或 Squid）
2. **配置监听本地端口**（如 7890）
3. **转发到香港服务器**（47.243.177.166:3128）

## 🔍 为什么豆包方法可能更有效

1. **Mac 作为中转**：
   - Mac 可以安装更强大的代理客户端（ClashX Pro）
   - 支持更多协议（HTTP/SOCKS5）
   - 更好的 HTTPS 处理

2. **局域网连接**：
   - iPhone 通过局域网连接 Mac
   - 更稳定、更快速
   - 避免直接连接服务器的网络问题

3. **更好的证书处理**：
   - ClashX Pro 可以更好地处理 HTTPS 证书
   - 避免浏览器安全策略问题
