# 📱 老版本Android USB调试指南

## 🔍 你看到的情况

你看到的是：
- 仅充电
- 传输文件
- 传输照片

这是**老版本Android**的USB连接模式选择。

---

## 🚀 解决方案

### 方案1：先启用开发者选项（重要！）

#### 步骤1：启用开发者模式

1. **设置 → 关于手机**
2. **找到"版本号"或"内部版本号"**
3. **连续点击7次**
   - 会提示"您已成为开发者"或"开发者模式已启用"

#### 步骤2：返回设置

1. **返回设置主页面**
2. **找到"开发者选项"**
   - 可能在"系统"或"关于手机"附近
   - 也可能在设置主页面直接显示

#### 步骤3：启用USB调试

1. **打开"开发者选项"**
2. **找到"USB调试"**
3. **启用它**

---

### 方案2：使用"传输文件"模式

#### 如果找不到开发者选项：

1. **选择"传输文件"模式**
2. **连接设备**
3. **尝试运行应用**：
   ```bash
   adb devices
   flutter run -d android
   ```

**注意**：某些老版本Android可能不支持USB调试，只能通过这种方式。

---

## 🔧 检查设备连接

### 步骤1：选择USB模式

在手机上选择：
- **"传输文件"** 或 **"MTP"** 模式

### 步骤2：检查连接

在终端运行：

```bash
adb devices
```

**应该看到**：
```
List of devices attached
<设备ID>    device
```

**如果显示"unauthorized"**：
- 在手机上确认"允许USB调试"（如果弹出）

**如果显示"offline"**：
- 尝试重新连接
- 或选择不同的USB模式

---

## ⚠️ 老版本Android的限制

### 可能的问题：

1. **Android版本太老**：
   - 如果Android版本低于5.0（API 21），可能不支持
   - Flutter要求Android 5.0+

2. **USB调试不可用**：
   - 某些老设备可能不支持USB调试
   - 只能通过其他方式安装应用

---

## 🎯 替代方案

### 方案1：检查Android版本

在手机上：
- 设置 → 关于手机 → Android版本

**如果Android版本低于5.0**：
- 无法运行Flutter应用
- 需要Android 5.0或更高版本

---

### 方案2：使用模拟器（如果网络允许）

如果真实设备太老，可以使用模拟器：
- 创建新版本的Android模拟器
- 测试应用

---

### 方案3：等自己的设备

如果设备太老或无法启用USB调试：
- 等自己的设备
- 或先完成其他工作

---

## 📋 步骤总结

### 现在请这样做：

1. **检查Android版本**：
   - 设置 → 关于手机 → Android版本
   - 如果是5.0以下，无法运行

2. **启用开发者选项**：
   - 设置 → 关于手机 → 连续点击"版本号"7次

3. **启用USB调试**：
   - 设置 → 开发者选项 → 启用"USB调试"

4. **选择USB模式**：
   - 选择"传输文件"或"MTP"

5. **检查连接**：
   ```bash
   adb devices
   ```

6. **运行应用**：
   ```bash
   flutter run -d android
   ```
