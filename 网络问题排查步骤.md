# 🔧 网络问题排查步骤

## ❌ 当前问题

应用显示示例内容，而不是您上传的真实视频。

## 📋 诊断步骤

### 步骤1：检查终端日志

在运行 `flutter run` 的终端窗口中，查找以下日志：

**正常情况应该看到：**
```
开始加载内容...
正在加载视频...
请求视频列表: http://47.243.177.166:8081/api/list/原创视频
视频加载完成，数量: 2
```

**如果有错误，可能看到：**
```
请求异常: ...
加载视频失败: ...
```

### 步骤2：在手机浏览器中测试

1. 打开手机 Safari 浏览器
2. 访问：`http://47.243.177.166:8081`
3. 查看是否能访问

**结果判断：**
- ✅ 如果浏览器能访问：说明网络正常，可能是应用配置问题
- ❌ 如果浏览器也无法访问：说明是网络问题（可能是运营商限制）

### 步骤3：检查网络连接

**可能的原因：**
1. **运营商限制**：某些运营商可能限制访问特定 IP
2. **防火墙**：iOS 设备或网络防火墙阻止连接
3. **网络环境**：使用移动数据可能无法访问服务器

## 🔧 解决方案

### 方案1：使用 WiFi 连接（如果当前使用移动数据）

切换到 WiFi 网络，然后重试。

### 方案2：检查服务器防火墙

确保服务器防火墙允许来自您手机 IP 的连接。

### 方案3：使用 VPN（如果网络被限制）

如果运营商限制了访问，可能需要使用 VPN。

### 方案4：查看详细错误信息

请告诉我终端日志中的具体错误信息，我可以帮您进一步排查。
