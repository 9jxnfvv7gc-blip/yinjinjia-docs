# 🔍 404 错误排查（2025-11-26）

## ❌ 问题

访问网页时提示 404 Not Found 错误。

---

## 🔍 需要确认的信息

1. **你在哪里访问的**？
   - Mac 浏览器？
   - iPhone Safari？
   - 其他？

2. **访问的是什么网址**？
   - Google？
   - 其他网站？

3. **ClashX Pro 是否已配置并运行**？

---

## ✅ 排查步骤

### 步骤1：检查 ClashX Pro 状态

1. **查看菜单栏的 ClashX Pro 图标**
2. **确认是否已启动**（图标应该显示在右上角）
3. **确认是否已选择配置**（hongkong）
4. **确认是否已设置为系统代理**（应该有勾选标记）

### 步骤2：测试 Mac 访问

1. **在 Mac 上打开浏览器**
2. **访问**：`https://www.google.com`
3. **看是否能打开**

### 步骤3：测试代理连接

在 Mac 终端运行：

```bash
curl -x http://47.243.177.166:3128 -I https://www.google.com
```

如果返回 200 或 302，说明代理可以访问 Google。

### 步骤4：检查 ClashX Pro 配置

1. **打开 ClashX Pro 偏好设置**
2. **确认"允许局域网连接"已勾选**
3. **确认端口设置**（HTTP: 7890, SOCKS5: 7891）

---

## 🔧 常见问题解决

### 问题1：Mac 访问 Google 显示 404

**可能原因**：
- ClashX Pro 未正确配置
- 代理连接失败
- 配置文件错误

**解决**：
1. 重新检查 ClashX Pro 配置
2. 重新选择配置并设置为系统代理
3. 测试代理连接

### 问题2：iPhone 访问 Google 显示 404

**可能原因**：
- iPhone 代理配置错误
- Mac 内网 IP 不正确
- ClashX Pro 未开启局域网共享

**解决**：
1. 检查 iPhone Wi-Fi 代理配置
2. 确认 Mac 内网 IP 正确
3. 确认 ClashX Pro 已开启"允许局域网连接"

### 问题3：访问其他网站显示 404

**可能原因**：
- 网站本身不存在
- 代理配置问题

**解决**：
1. 尝试访问其他网站（如 baidu.com）
2. 检查代理配置
