# 服务器启动失败排查

## ❌ 问题现象

- ❌ 服务器启动失败：`status=1/FAILURE`
- ⏳ 需要检查错误原因

---

## 🔍 排查步骤

### 步骤1：查看详细错误日志

在Workbench终端执行：

```bash
# 查看最近的错误日志
journalctl -u video-server -n 50 --no-pager
```

**应该显示**：具体的错误信息（语法错误、导入错误等）

---

### 步骤2：检查语法

```bash
# 检查Python语法
python3 -m py_compile /root/video_server/video_server.py
```

**如果有错误**，会显示具体的错误信息和行号。

---

### 步骤3：手动测试运行

```bash
# 手动运行服务器，查看错误
cd /root/video_server
python3 video_server.py
```

**应该显示**：具体的错误信息

---

### 步骤4：检查文件完整性

```bash
# 检查文件是否有语法问题
python3 -c "import ast; ast.parse(open('/root/video_server/video_server.py').read())"
```

---

## 🔧 可能的问题

### 问题1：语法错误

**检查**：`python3 -m py_compile` 的输出

**解决方案**：修复语法错误

---

### 问题2：缩进错误

**检查**：Python缩进是否正确

**解决方案**：修复缩进

---

### 问题3：导入错误

**检查**：是否缺少必要的模块

**解决方案**：安装缺失的模块

---

3. **手动测试运行**（`cd /root/video_server && python3 video_server.py`）
